title: "Quickstart"
description: "Get from zero to storing your first secret in Vault in under five minutes."
---

This guide walks you through installing Vault, starting a development server, and storing and retrieving your first secret — using the CLI, the HTTP API, and the Go SDK.

<Steps>
  <Step title="Install Vault">
    <CodeGroup>

    ```bash macOS (Homebrew)
    brew tap hashicorp/tap
    brew install hashicorp/tap/vault
    ```

    ```bash Linux (apt)
    wget -O - https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
    echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
    sudo apt update && sudo apt install vault
    ```

    ```bash Linux (yum)
    sudo yum install -y yum-utils
    sudo yum-config-manager --add-repo https://rpm.releases.hashicorp.com/RHEL/hashicorp.repo
    sudo yum -y install vault
    ```

    </CodeGroup>

    Verify the installation:

    ```bash
    vault version
    ```

    ```
    Vault v1.22.0-beta1
    ```
  </Step>

  <Step title="Start Vault in dev mode">
    Dev mode starts Vault in-memory with a pre-configured root token. It is unsealed automatically and ready to use immediately.

    ```bash
    vault server -dev -dev-root-token-id="root"
    ```

    You will see output similar to:

    ```
    ==> Vault server configuration:

                 Api Address: http://127.0.0.1:8200
                         Cgo: disabled
             Cluster Address: https://127.0.0.1:8201
                  Go Version: go1.22.0
                  Log Level: info
                      Mlock: supported: false, enabled: false
              Recovery Mode: false
                    Storage: inmem
                    Version: Vault v1.22.0-beta1

    ==> Vault server started! Log data will stream in below:
    ```

    <Warning>
      Dev mode stores all data in memory. Everything is lost when the process stops. Never use dev mode in production.
    </Warning>

    Leave this terminal running and open a new terminal for the next steps.
  </Step>

  <Step title="Set environment variables">
    Tell the Vault CLI where to find the server and which token to use:

    ```bash
    export VAULT_ADDR='http://127.0.0.1:8200'
    export VAULT_TOKEN='root'
    ```

    Confirm Vault is reachable:

    ```bash
    vault status
    ```

    ```
    Key             Value
    ---             -----
    Seal Type       shamir
    Initialized     true
    Sealed          false
    Total Shares    1
    Threshold       1
    Version         1.22.0-beta1
    Storage Type    inmem
    Cluster Name    vault-cluster-...
    HA Enabled      false
    ```

    The exit code from `vault status` reflects the seal state: `0` = unsealed, `2` = sealed.
  </Step>

  <Step title="Store a secret">
    Write a key/value secret to the `secret/` KV v2 mount that dev mode creates automatically:

    ```bash
    vault kv put secret/hello foo=world
    ```

    ```
    ======== Secret Path ========
    secret/data/hello

    ======= Metadata =======
    Key                Value
    ---                -----
    created_time       2024-01-15T10:00:00.000000000Z
    custom_metadata    <nil>
    deletion_time      n/a
    destroyed          false
    version            1
    ```

    You can store multiple key/value pairs in a single secret:

    ```bash
    vault kv put secret/myapp/config \
      db_host=postgres.internal \
      db_port=5432 \
      db_name=myapp
    ```
  </Step>

  <Step title="Read a secret">
    ```bash
    vault kv get secret/hello
    ```

    ```
    ======== Secret Path ========
    secret/data/hello

    ======= Metadata =======
    Key                Value
    ---                -----
    created_time       2024-01-15T10:00:00.000000000Z
    custom_metadata    <nil>
    deletion_time      n/a
    destroyed          false
    version            1

    === Data ===
    Key    Value
    ---    -----
    foo    world
    ```

    To retrieve only a specific field:

    ```bash
    vault kv get -field=foo secret/hello
    ```

    ```
    world
    ```

    To retrieve a specific version:

    ```bash
    vault kv get -version=1 secret/hello
    ```
  </Step>

  <Step title="Use the HTTP API">
    Vault exposes a full REST API. Every CLI command corresponds to an API call.

    **Write a secret:**

    ```bash
    curl \
      --header "X-Vault-Token: root" \
      --request POST \
      --data '{"data": {"foo": "world"}}' \
      http://127.0.0.1:8200/v1/secret/data/hello
    ```

    **Read a secret:**

    ```bash
    curl \
      --header "X-Vault-Token: root" \
      http://127.0.0.1:8200/v1/secret/data/hello
    ```

    ```json
    {
      "request_id": "...",
      "data": {
        "data": {
          "foo": "world"
        },
        "metadata": {
          "created_time": "2024-01-15T10:00:00Z",
          "version": 1
        }
      }
    }
    ```

    **Check seal status (no token required):**

    ```bash
    curl http://127.0.0.1:8200/v1/sys/health
    ```

    <Tip>
      Set `VAULT_FORMAT=json` to get JSON output from any CLI command, which is useful for scripting.
    </Tip>
  </Step>

  <Step title="Use the Go SDK">
    Install the Vault API library:

    ```bash
    go get github.com/hashicorp/vault/api
    ```

    The following example uses the `KVv2` client from `api/kv_v2.go`. The `KVv2.Put` method writes data, and `KVv2.Get` retrieves it:

    ```go
    package main

    import (
      "context"
      "fmt"
      "log"

      vault "github.com/hashicorp/vault/api"
    )

    func main() {
      config := vault.DefaultConfig()
      config.Address = "http://127.0.0.1:8200"

      client, err := vault.NewClient(config)
      if err != nil {
        log.Fatalf("unable to initialize Vault client: %v", err)
      }

      client.SetToken("root")

      ctx := context.Background()

      // Write a secret using KVv2.Put
      // Put inserts a key-value secret into the KV v2 secrets engine.
      // If the secret already exists, a new version will be created.
      secretData := map[string]interface{}{
        "foo": "world",
      }

      kv := client.KVv2("secret")

      _, err = kv.Put(ctx, "hello", secretData)
      if err != nil {
        log.Fatalf("unable to write secret: %v", err)
      }
      fmt.Println("Secret written successfully.")

      // Read the secret back using KVv2.Get
      // Get returns the latest version of a secret from the KV v2 secrets engine.
      secret, err := kv.Get(ctx, "hello")
      if err != nil {
        log.Fatalf("unable to read secret: %v", err)
      }

      value, ok := secret.Data["foo"].(string)
      if !ok {
        log.Fatal("value type assertion failed")
      }

      fmt.Printf("The secret value for 'foo' is: %s\n", value)

      // Retrieve a specific version using KVv2.GetVersion
      versionedSecret, err := kv.GetVersion(ctx, "hello", 1)
      if err != nil {
        log.Fatalf("unable to read secret version: %v", err)
      }
      fmt.Printf("Version 1 created at: %s\n", versionedSecret.VersionMetadata.CreatedTime)
    }
    ```

    Key methods on `KVv2` from the SDK (`api/kv_v2.go`):

    | Method | Description |
    |--------|-------------|
    | `Put(ctx, path, data, opts...)` | Write or overwrite a secret; creates a new version if the secret exists |
    | `Get(ctx, path)` | Read the latest version of a secret |
    | `GetVersion(ctx, path, version)` | Read a specific version of a secret |
    | `GetVersionsAsList(ctx, path)` | List all version metadata sorted by version number |
    | `GetMetadata(ctx, path)` | Read full metadata including all versions |
    | `Patch(ctx, path, newData, opts...)` | Additively update a secret without overwriting unchanged keys |
    | `Delete(ctx, path)` | Soft-delete the latest version |
    | `DeleteVersions(ctx, path, versions)` | Soft-delete specific versions |
    | `Destroy(ctx, path, versions)` | Permanently remove secret data for specified versions |
    | `Rollback(ctx, path, toVersion)` | Promote a previous version to become the latest version |
  </Step>
</Steps>
