---
title: "vault status"
description: "Print the current seal and HA status of a Vault server, regardless of whether it is sealed."
---

## Overview

The `vault status` command prints the current state of Vault including whether it is sealed and if HA mode is enabled. This command prints output regardless of whether Vault is sealed.

```
Usage: vault status [options]
```

## Exit Codes

The exit code from `vault status` reflects the seal state:

| Exit Code | Meaning |
|---|---|
| `0` | Vault is unsealed |
| `1` | An error occurred |
| `2` | Vault is sealed |

<Tip>
  Use the exit code in scripts to check seal state without parsing output:

  ```bash
  vault status > /dev/null 2>&1
  if [ $? -eq 2 ]; then
    echo "Vault is sealed"
  fi
  ```
</Tip>

## Flags

<ParamField path="-format" type="string" default="table">
  Output format. One of `table`, `json`, or `yaml`.
</ParamField>

## Examples

```bash
# Default table output
vault status

# JSON output (useful for scripting)
vault status -format=json

# YAML output
vault status -format=yaml

# Check if Vault is unsealed (exit code 0 = unsealed)
vault status && echo "Vault is ready"
```

<Note>
  `vault status` always queries the root namespace regardless of any `-namespace` flag or `VAULT_NAMESPACE` environment variable, because the seal status endpoint is only available in the root namespace before Vault is unsealed.
</Note>
