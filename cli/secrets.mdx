title: "vault secrets"

---

## Overview

The `vault secrets` command groups subcommands for managing secrets engines.

```
Usage: vault secrets <subcommand> [options] [args]
```

| Subcommand | Description |
|---|---|
| `enable` | Enable a secrets engine |
| `disable` | Disable a secrets engine |
| `list` | List enabled secrets engines |
| `move` | Move a secrets engine to a new path |
| `tune` | Tune a secrets engine configuration |

---

## vault secrets enable

Enables a secrets engine at the given path. By default, secrets engines are enabled at the path corresponding to their TYPE.

```
Usage: vault secrets enable [options] TYPE
```

### Flags

<ParamField path="-path" type="string">
  Place where the secrets engine will be accessible. Must be unique across all secrets engines. Defaults to the TYPE of the secrets engine.
</ParamField>

<ParamField path="-description" type="string">
  Human-friendly description for the purpose of this engine.
</ParamField>

<ParamField path="-default-lease-ttl" type="duration">
  The default lease TTL for this secrets engine. If unspecified, defaults to the Vault server's globally configured default lease TTL.
</ParamField>

<ParamField path="-max-lease-ttl" type="duration">
  The maximum lease TTL for this secrets engine. If unspecified, defaults to the Vault server's globally configured maximum lease TTL.
</ParamField>

<ParamField path="-audit-non-hmac-request-keys" type="string">
  Key that will not be HMAC'd by audit devices in the request data object. Can be specified multiple times.
</ParamField>

<ParamField path="-audit-non-hmac-response-keys" type="string">
  Key that will not be HMAC'd by audit devices in the response data object. Can be specified multiple times.
</ParamField>

<ParamField path="-listing-visibility" type="string">
  Determines the visibility of the mount in the UI-specific listing endpoint.
</ParamField>

<ParamField path="-passthrough-request-headers" type="string">
  Request header value that will be sent to the plugin. Can be specified multiple times.
</ParamField>

<ParamField path="-allowed-response-headers" type="string">
  Response header value that plugins will be allowed to set. Can be specified multiple times.
</ParamField>

<ParamField path="-force-no-cache" type="bool" default="false">
  Force the secrets engine to disable caching. Does not affect caching of the underlying encrypted data storage.
</ParamField>

<ParamField path="-plugin-name" type="string">
  Name of the secrets engine plugin. This plugin name must already exist in Vault's plugin catalog.
</ParamField>

<ParamField path="-plugin-version" type="string">
  Select the semantic version of the plugin to enable.
</ParamField>

<ParamField path="-options" type="string">
  Key-value pair provided as `key=value` for mount options. Can be specified multiple times. For the KV secrets engine, use `-options version=2` to enable KV v2.
</ParamField>

<ParamField path="-local" type="bool" default="false">
  Mark the secrets engine as local-only. Local engines are not replicated or removed by replication.
</ParamField>

<ParamField path="-seal-wrap" type="bool" default="false">
  Enable seal wrapping of critical values in the secrets engine.
</ParamField>

<ParamField path="-external-entropy-access" type="bool" default="false">
  Enable secrets engine to access Vault's external entropy source.
</ParamField>

<ParamField path="-version" type="int" default="0">
  Select the version of the engine to run. Not supported by all engines. Sets `options[version]` under the hood.
</ParamField>

<ParamField path="-allowed-managed-keys" type="string">
  Managed key name(s) that this mount is allowed to access. Can be specified multiple times.
</ParamField>

<ParamField path="-delegated-auth-accessors" type="string">
  A list of permitted authentication accessors this backend can delegate authentication to. Can be specified multiple times.
</ParamField>

<ParamField path="-identity-token-key" type="string" default="default">
  Select the key used to sign plugin identity tokens.
</ParamField>

<ParamField path="-override-pinned-version" type="bool">
  Enterprise only. The specified `-plugin-version` will override the pinned plugin version.
</ParamField>

### Examples

```bash
# Enable the AWS secrets engine at aws/
vault secrets enable aws

# Enable the SSH secrets engine at a custom path
vault secrets enable -path=ssh-prod ssh

# Enable KV v1
vault secrets enable -path=secret kv

# Enable KV v2
vault secrets enable -path=secret -version=2 kv

# Enable KV v2 using options
vault secrets enable -path=secret -options version=2 kv

# Enable with a custom max TTL
vault secrets enable -max-lease-ttl=30m database

# Enable a custom plugin
vault secrets enable -path=my-secrets my-plugin
```

## vault secrets disable

Disables a secrets engine at the given path. All secrets created by this engine are revoked and its Vault data is removed.

```
Usage: vault secrets disable [options] PATH
```

<Warning>
  Disabling a secrets engine revokes all secrets it has created. This action is destructive and cannot be undone.
</Warning>

### Examples

```bash
vault secrets disable aws/
vault secrets disable pki/
```

## vault secrets list

Lists the enabled secrets engines on the Vault server.

```
Usage: vault secrets list [options]
```

### Flags

<ParamField path="-detailed" type="bool" default="false">
  Print detailed information such as TTLs and replication status about each secrets engine.
</ParamField>

### Examples

```bash
# List all enabled secrets engines
vault secrets list

# List with detailed output
vault secrets list -detailed

# JSON output
vault secrets list -format=json
```

### Output Columns (detailed)

When `-detailed` is used, the table includes: Path, Plugin, Accessor, Default TTL, Max TTL, Force No Cache, Replication, Seal Wrap, External Entropy Access, Options, Description, UUID, Version, Running Version, Running SHA256, Deprecation Status.

---

## vault secrets move

Moves an existing secrets engine to a new path. This effectively disables the engine at the old path and re-enables it at the new path, preserving all data.

```
Usage: vault secrets move [options] SOURCE DESTINATION
```

### Examples

```bash
vault secrets move secret/ kv-old/
```

## vault secrets tune

Tunes the configuration options for the secrets engine at the given path.

```
Usage: vault secrets tune [options] PATH
```

### Flags

<ParamField path="-default-lease-ttl" type="duration" default="0">
  The default lease TTL for this secrets engine. Defaults to the Vault server's globally configured value or a previously configured value.
</ParamField>

<ParamField path="-max-lease-ttl" type="duration" default="0">
  The maximum lease TTL for this secrets engine.
</ParamField>

<ParamField path="-description" type="string">
  Human-friendly description of this secrets engine. Overrides the current stored value.
</ParamField>

<ParamField path="-audit-non-hmac-request-keys" type="string">
  Key that will not be HMAC'd by audit devices in the request data object. Can be specified multiple times.
</ParamField>

<ParamField path="-audit-non-hmac-response-keys" type="string">
  Key that will not be HMAC'd by audit devices in the response data object. Can be specified multiple times.
</ParamField>

<ParamField path="-listing-visibility" type="string">
  Determines the visibility of the mount in the UI-specific listing endpoint.
</ParamField>

<ParamField path="-passthrough-request-headers" type="string">
  Request header value that will be sent to the plugin. Can be specified multiple times.
</ParamField>

<ParamField path="-allowed-response-headers" type="string">
  Response header value that plugins will be allowed to set. Can be specified multiple times.
</ParamField>

<ParamField path="-options" type="string">
  Key-value pair provided as `key=value` for mount options. Can be specified multiple times.
</ParamField>

<ParamField path="-version" type="int" default="0">
  Select the version of the engine to run. Not supported by all engines.
</ParamField>

<ParamField path="-plugin-version" type="string">
  Select the semantic version of the plugin to run. The new version must be registered in the plugin catalog, and will not start running until the plugin is reloaded.
</ParamField>

<ParamField path="-allowed-managed-keys" type="string">
  Managed key name(s) that this mount is allowed to access. Can be specified multiple times.
</ParamField>

<ParamField path="-delegated-auth-accessors" type="string">
  A list of permitted authentication accessors this backend can delegate authentication to. Can be specified multiple times.
</ParamField>

<ParamField path="-identity-token-key" type="string" default="default">
  Select the key used to sign plugin identity tokens.
</ParamField>

<ParamField path="-override-pinned-version" type="bool">
  Enterprise only. The specified `-plugin-version` will override the pinned plugin version.
</ParamField>

### Examples

```bash
# Set the default lease TTL for the PKI engine
vault secrets tune -default-lease-ttl=72h pki/

# Update the description
vault secrets tune -description="Production KV store" secret/

# Tune the max lease TTL
vault secrets tune -max-lease-ttl=720h database/
```
