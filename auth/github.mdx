---
title: "GitHub Auth Method"
description: "The GitHub auth method allows users to authenticate with Vault using their GitHub personal access tokens."
---

The GitHub auth method allows members of a GitHub organization to authenticate with Vault using a GitHub personal access token. Policies are assigned based on GitHub team membership and optionally per-user.

## Enabling GitHub auth

```bash
vault auth enable github
```

## Configuration

```bash
vault write auth/github/config \
  organization=my-github-org \
  token_ttl="1h" \
  token_max_ttl="24h"
```

### Configuration parameters

<ParamField path="organization" type="string" required>
  The GitHub organization users must be a member of to authenticate. This is the organization login name (e.g., `hashicorp`, not the full URL).
</ParamField>

<ParamField path="organization_id" type="number">
  The numeric ID of the organization users must be a member of. If not provided, Vault will look it up automatically using the GitHub API on first login. Setting this prevents renaming attacks.
</ParamField>

<ParamField path="base_url" type="string">
  The API endpoint to use. Useful if you are running GitHub Enterprise or an API-compatible authentication server. Example: `https://github.example.com/api/v3/`.
</ParamField>

<ParamField path="token_ttl" type="string">
  The incremental lifetime for generated tokens. Defaults to system default if unset.
</ParamField>

<ParamField path="token_max_ttl" type="string">
  The maximum lifetime for generated tokens. Defaults to system default if unset.
</ParamField>

<ParamField path="token_policies" type="string[]">
  List of policies applied to all tokens generated by this auth method, in addition to any team- or user-specific policies.
</ParamField>

<ParamField path="token_bound_cidrs" type="string[]">
  List of CIDR blocks. If set, restricts which IP addresses can authenticate and use the resulting token.
</ParamField>

<ParamField path="token_period" type="string">
  If set, tokens will be periodic with the given period.
</ParamField>

<ParamField path="token_type" type="string" default="default">
  The type of token to generate: `service`, `batch`, or `default`.
</ParamField>

## Mapping teams to policies

Policies are assigned to users based on their GitHub team memberships. Both the team name and the team slug are checked.

```bash
# Map a team to one or more policies
vault write auth/github/map/teams/engineering \
  value=engineering-policy,read-secrets

# Map another team
vault write auth/github/map/teams/ops \
  value=ops-policy

# Read a team mapping
vault read auth/github/map/teams/engineering

# List all team mappings
vault list auth/github/map/teams
```

## Mapping users to policies

You can also assign policies to specific GitHub users:

```bash
vault write auth/github/map/users/octocat \
  value=admin-policy
```

## Logging in

Users authenticate using a GitHub personal access token. The token must have the `read:org` scope to verify organization membership.

```bash
vault login -method=github token=<github-personal-access-token>
```

Or set the token via environment variable:

```bash
export VAULT_AUTH_GITHUB_TOKEN="github_pat_..."
vault login -method=github
```

### Creating a GitHub personal access token

The GitHub token requires the following scopes:

- `read:org` — to verify organization and team membership

<Steps>
  <Step title="Go to GitHub Settings">
    Navigate to **Settings > Developer Settings > Personal access tokens > Tokens (classic)**.
  </Step>
  <Step title="Generate a new token">
    Click **Generate new token** and select the `read:org` scope.
  </Step>
  <Step title="Copy the token">
    Copy the generated token immediately — GitHub will not show it again.
  </Step>
  <Step title="Use the token with Vault">
    ```bash
    vault login -method=github token=ghp_xxxxxxxxxxxx
    ```
  </Step>
</Steps>

## GitHub Enterprise

To use GitHub Enterprise, set the `base_url` to point to your GitHub Enterprise API:

```bash
vault write auth/github/config \
  organization=my-org \
  base_url="https://github.example.com/api/v3/"
```

<Note>
  The `base_url` must end with a trailing slash and point to the API endpoint, not the GitHub web interface.
</Note>

## Example: Full setup

```bash
# 1. Enable
vault auth enable github

# 2. Configure the organization
vault write auth/github/config organization=acmecorp

# 3. Map the engineering team to a policy
vault write auth/github/map/teams/engineering value=dev-policy

# 4. Map the platform team to an admin policy
vault write auth/github/map/teams/platform value=admin-policy

# 5. A member of the engineering team logs in
vault login -method=github token=ghp_xxxxxx
```
