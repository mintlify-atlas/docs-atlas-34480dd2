---
title: "LDAP Auth Method"
description: "The LDAP auth method allows authentication using an existing LDAP server, including Active Directory."
---

The LDAP auth method authenticates users against an LDAP directory. It supports both OpenLDAP and Microsoft Active Directory. After authentication, LDAP group membership is used to assign Vault policies.

## Enabling LDAP auth

```bash
vault auth enable ldap
```

## Configuration

```bash
vault write auth/ldap/config \
  url="ldaps://ldap.example.com" \
  userdn="ou=Users,dc=example,dc=com" \
  groupdn="ou=Groups,dc=example,dc=com" \
  binddn="cn=vault,dc=example,dc=com" \
  bindpass="vaultbindpassword" \
  userattr="uid"
```

### Configuration parameters

#### Connection

<ParamField path="url" type="string" default="ldap://127.0.0.1">
  LDAP URL to connect to. Use `ldap://` for plain text (port 389) and `ldaps://` for TLS (port 636). Multiple URLs can be specified separated by commas; they will be tried in order.
</ParamField>

<ParamField path="starttls" type="boolean" default="false">
  If true, issues a `StartTLS` command after connecting. Only applicable when using `ldap://` (not `ldaps://`).
</ParamField>

<ParamField path="tls_min_version" type="string" default="tls12">
  Minimum TLS version to use. Accepted values: `tls10`, `tls11`, `tls12`, `tls13`.
</ParamField>

<ParamField path="tls_max_version" type="string" default="tls12">
  Maximum TLS version to use.
</ParamField>

<ParamField path="insecure_tls" type="boolean" default="false">
  If true, skips LDAP server SSL certificate verification. Not recommended for production.
</ParamField>

<ParamField path="certificate" type="string">
  CA certificate to use when verifying LDAP server certificate. Must be x509 PEM encoded.
</ParamField>

#### Bind credentials

<ParamField path="binddn" type="string">
  LDAP distinguished name for searching for the user DN (optional). Example: `cn=vault,dc=example,dc=com`.
</ParamField>

<ParamField path="bindpass" type="string">
  LDAP password for the bind DN. Used when searching for the user DN.
</ParamField>

#### User search

<ParamField path="userdn" type="string">
  LDAP domain to use for user searches. Example: `ou=People,dc=example,dc=org`.
</ParamField>

<ParamField path="userattr" type="string" default="cn">
  The LDAP attribute to use as the username when performing user lookups. For OpenLDAP, this is typically `uid`. For Active Directory, this is typically `sAMAccountName`.
</ParamField>

<ParamField path="userfilter" type="string">
  Go template for a custom user search filter. Can reference `{{.UserAttr}}` and `{{.Username}}`.
</ParamField>

<ParamField path="discoverdn" type="boolean" default="false">
  If true, use an anonymous bind to discover the user DN before binding as the user.
</ParamField>

<ParamField path="deny_null_bind" type="boolean" default="true">
  If true, prevents unauthenticated access to the LDAP server when an empty password is provided. Strongly recommended.
</ParamField>

#### Group search

<ParamField path="groupdn" type="string">
  LDAP search base to use for group membership searches. Example: `ou=Groups,dc=example,dc=org`.
</ParamField>

<ParamField path="groupattr" type="string" default="cn">
  LDAP attribute to extract group names from objects returned by `groupfilter`. Examples: `cn`, `memberOf`.
</ParamField>

<ParamField path="groupfilter" type="string" default="(|(memberUid={{.Username}})(member={{.UserDN}})(uniqueMember={{.UserDN}}))">
  Go template for querying group membership. The template can access `UserDN` and `Username` context variables.
</ParamField>

<ParamField path="anonymous_group_search" type="boolean" default="false">
  Use anonymous binds when performing LDAP group searches. The initial credentials will still be used for the initial connection test.
</ParamField>

#### Token parameters

<ParamField path="token_policies" type="string[]">
  List of policies applied to all tokens generated by this auth method, in addition to any group- or user-specific policies.
</ParamField>

<ParamField path="token_ttl" type="string">
  The incremental lifetime for generated tokens.
</ParamField>

<ParamField path="token_max_ttl" type="string">
  The maximum lifetime for generated tokens.
</ParamField>

<ParamField path="token_bound_cidrs" type="string[]">
  List of CIDR blocks restricting which IPs can use the returned token.
</ParamField>

## Group policy mapping

After configuring the LDAP server, map LDAP groups to Vault policies:

```bash
# Map an LDAP group to a policy
vault write auth/ldap/groups/engineering policies=engineering-policy

# Map multiple policies
vault write auth/ldap/groups/ops policies=ops-policy,secrets-read

# List configured groups
vault list auth/ldap/groups

# Read a group mapping
vault read auth/ldap/groups/engineering
```

## User policy mapping

You can also assign policies directly to individual users, supplementing their group policies:

```bash
vault write auth/ldap/users/mitchellh policies=admin-policy
```

## Logging in

```bash
vault login -method=ldap username=mitchellh password=foo
```

Or using environment variables:

```bash
export VAULT_AUTH_LDAP_USERNAME=mitchellh
vault login -method=ldap
```

## Active Directory configuration

Active Directory uses different attribute names:

```bash
vault write auth/ldap/config \
  url="ldaps://ad.example.com" \
  userdn="CN=Users,DC=example,DC=com" \
  userattr="sAMAccountName" \
  groupdn="CN=Groups,DC=example,DC=com" \
  groupattr="cn" \
  groupfilter="(&(objectClass=group)(member:1.2.840.113556.1.4.1941:={{.UserDN}}))" \
  binddn="CN=VaultService,CN=Users,DC=example,DC=com" \
  bindpass="ServiceAccountPassword" \
  starttls=true
```

## Root password rotation

Vault can automatically rotate the LDAP bind password:

```bash
# Configure a password policy
vault write sys/policies/password/ldap-policy policy='{
  length = 20
  rule "charset" {
    charset = "abcdefghijklmnopqrstuvwxyz"
    min-chars = 1
  }
}'

# Set the password policy on the LDAP config
vault write auth/ldap/config \
  password_policy=ldap-policy \
  ...

# Rotate the password
vault write -f auth/ldap/rotate-root
```

<Warning>
  After rotating the root password, ensure the new password is updated in the LDAP directory before the next rotation to avoid lockouts.
</Warning>

## Example: Full OpenLDAP setup

```bash
# 1. Enable
vault auth enable ldap

# 2. Configure
vault write auth/ldap/config \
  url="ldap://ldap.example.com" \
  userdn="ou=Users,dc=example,dc=com" \
  groupdn="ou=Groups,dc=example,dc=com" \
  groupattr="cn" \
  userattr="uid" \
  binddn="cn=vault-service,dc=example,dc=com" \
  bindpass="securepassword" \
  starttls=true

# 3. Map groups to policies
vault write auth/ldap/groups/developers policies=dev-policy
vault write auth/ldap/groups/operators policies=ops-policy

# 4. Log in
vault login -method=ldap username=jsmith password=mypassword
```
