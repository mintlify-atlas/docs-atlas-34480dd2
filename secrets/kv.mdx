---
title: "KV Secrets Engine"
description: "The KV secrets engine stores arbitrary key-value pairs in Vault, with optional versioning in v2."
---

The KV (key-value) secrets engine stores arbitrary secrets within the configured physical storage for Vault. It is the simplest secrets engine and is useful for storing sensitive configuration, credentials, and other small pieces of data.

## KV v1 vs KV v2

<Tabs>
  <Tab title="KV v1">
    KV v1 stores key-value pairs without versioning. Writing to an existing key overwrites the previous value permanently.

    ```bash
    vault secrets enable -path=secret kv
    ```

    **Characteristics:**
    - No versioning — previous values are overwritten and unrecoverable
    - Simpler data model: `{key: value}` at a path
    - Slightly lower read overhead
    - Suitable for rarely-changing, non-sensitive configuration
  </Tab>
  <Tab title="KV v2">
    KV v2 adds full versioning, soft-delete, metadata management, and check-and-set operations.

    ```bash
    vault secrets enable -path=secret -version=2 kv
    # or
    vault secrets enable kv-v2
    ```

    **Characteristics:**
    - Each write creates a new version; previous versions are retained
    - Soft-delete: deleted versions can be undeleted
    - Destroy: permanently removes a version's data
    - Metadata: configurable `max_versions`, `cas_required`, `delete_version_after`, `custom_metadata`
    - Check-and-set (CAS) prevents blind writes

    <Tip>
      KV v2 is the recommended version for new deployments. It provides version history and protection against accidental overwrites.
    </Tip>
  </Tab>
</Tabs>

## Enabling KV v2

```bash
vault secrets enable -path=secret kv-v2
```

To upgrade an existing KV v1 mount to v2:

```bash
vault kv enable-versioning secret/
```

## Basic operations

<Steps>
  <Step title="Write a secret">
    ```bash
    vault kv put secret/myapp username=admin password=s3cr3t
    ```

    With multiple fields:

    ```bash
    vault kv put secret/myapp \
      username=admin \
      password=s3cr3t \
      host=db.example.com \
      port=5432
    ```

    Via the API:

    ```bash
    curl \
      --header "X-Vault-Token: $VAULT_TOKEN" \
      --request POST \
      --data '{"data":{"username":"admin","password":"s3cr3t"}}' \
      $VAULT_ADDR/v1/secret/data/myapp
    ```
  </Step>

  <Step title="Read a secret">
    ```bash
    vault kv get secret/myapp
    ```

    To output only a specific field:

    ```bash
    vault kv get -field=password secret/myapp
    ```

    Via the API:

    ```bash
    curl \
      --header "X-Vault-Token: $VAULT_TOKEN" \
      $VAULT_ADDR/v1/secret/data/myapp
    ```
  </Step>

  <Step title="List secrets">
    ```bash
    vault kv list secret/
    ```

    Lists keys at the given prefix. Note that KV v2 stores data at `secret/data/<path>`, but the CLI handles path normalization automatically.
  </Step>

  <Step title="Delete a secret">
    In KV v2, deleting a secret creates a soft-delete — the version is marked deleted but its metadata and data are retained.

    ```bash
    vault kv delete secret/myapp
    ```

    To delete specific versions:

    ```bash
    vault kv delete -versions=1,2 secret/myapp
    ```
  </Step>
</Steps>

## Versioning in KV v2

Every write to a KV v2 path creates a new version. Vault retains all versions up to `max_versions` (default: unlimited until storage runs out).

### Read a specific version

```bash
vault kv get -version=2 secret/myapp
```

Via the API:

```bash
curl \
  --header "X-Vault-Token: $VAULT_TOKEN" \
  "$VAULT_ADDR/v1/secret/data/myapp?version=2"
```

### Undelete a version

Restore a soft-deleted version:

```bash
vault kv undelete -versions=2 secret/myapp
```

Via the API:

```bash
curl \
  --header "X-Vault-Token: $VAULT_TOKEN" \
  --request POST \
  --data '{"versions":[2]}' \
  $VAULT_ADDR/v1/secret/undelete/myapp
```

### Destroy a version

Permanently remove the data for a version. Unlike delete, destroy cannot be undone.

<Warning>
  Destroying a version permanently removes its data. The version metadata (creation time, deletion time) remains, but the secret values are gone forever.
</Warning>

```bash
vault kv destroy -versions=1 secret/myapp
```

Via the API:

```bash
curl \
  --header "X-Vault-Token: $VAULT_TOKEN" \
  --request POST \
  --data '{"versions":[1]}' \
  $VAULT_ADDR/v1/secret/destroy/myapp
```

## Metadata management

KV v2 stores metadata separately from secret data. The metadata path is `secret/metadata/<path>`.

### View metadata

```bash
vault kv metadata get secret/myapp
```

The response includes:

| Field | Description |
|---|---|
| `current_version` | The latest version number |
| `oldest_version` | The oldest version still retained |
| `max_versions` | Maximum number of versions to retain |
| `cas_required` | Whether check-and-set is required for writes |
| `delete_version_after` | Duration after which versions are automatically deleted |
| `custom_metadata` | User-defined metadata key-value pairs |
| `versions` | Map of version number to version metadata |

### Update metadata

```bash
vault kv metadata put \
  -max-versions=10 \
  -delete-version-after=168h \
  -custom-metadata=team=platform \
  -custom-metadata=owner=alice \
  secret/myapp
```

<ParamField path="max_versions" type="number" default="0">
  Maximum number of versions to retain. When the limit is reached, the oldest version is removed. `0` means unlimited.
</ParamField>

<ParamField path="cas_required" type="boolean" default="false">
  Requires the `cas` parameter on all write requests, preventing blind overwrites.
</ParamField>

<ParamField path="delete_version_after" type="string" default="0s">
  Duration string (e.g., `168h`) after which versions are automatically soft-deleted.
</ParamField>

<ParamField path="custom_metadata" type="object">
  User-defined key-value pairs attached to the secret metadata.
</ParamField>

## Check-and-set (CAS)

CAS prevents accidental overwrites by requiring the write request to specify the current version. Vault rejects the write if the current version doesn't match.

```bash
# Only write if the current version is 3
vault kv put -cas=3 secret/myapp password=newvalue
```

## Go SDK examples

The Vault Go SDK provides typed `KVv1` and `KVv2` clients. Both are accessible from the main `Client` struct.

<CodeGroup>

```go kv_v1_example.go
package main

import (
    "context"
    "fmt"
    vault "github.com/hashicorp/vault/api"
)

func main() {
    client, _ := vault.NewClient(vault.DefaultConfig())
    client.SetToken("s.mytoken")

    kv := client.KVv1("secret")

    // Write a secret
    err := kv.Put(context.Background(), "myapp", map[string]interface{}{
        "username": "admin",
        "password": "s3cr3t",
    })

    // Read a secret
    secret, err := kv.Get(context.Background(), "myapp")
    if err != nil {
        panic(err)
    }
    fmt.Printf("password: %s\n", secret.Data["password"])

    // Delete a secret
    err = kv.Delete(context.Background(), "myapp")
}
```

```go kv_v2_example.go
package main

import (
    "context"
    "fmt"
    vault "github.com/hashicorp/vault/api"
)

func main() {
    client, _ := vault.NewClient(vault.DefaultConfig())
    client.SetToken("s.mytoken")

    kv := client.KVv2("secret")

    // Write creates a new version
    secret, err := kv.Put(context.Background(), "myapp", map[string]interface{}{
        "username": "admin",
        "password": "s3cr3t",
    })
    if err != nil {
        panic(err)
    }
    fmt.Printf("version: %d\n", secret.VersionMetadata.Version)

    // Read the latest version
    secret, err = kv.Get(context.Background(), "myapp")

    // Read a specific version
    secret, err = kv.GetVersion(context.Background(), "myapp", 1)

    // Get all version metadata
    versions, err := kv.GetVersionsAsList(context.Background(), "myapp")
    for _, v := range versions {
        fmt.Printf("version %d created at %s\n", v.Version, v.CreatedTime)
    }

    // Patch (partial update) — only updates the fields provided
    secret, err = kv.Patch(context.Background(), "myapp", map[string]interface{}{
        "password": "newpassword",
    })

    // Soft-delete the latest version
    err = kv.Delete(context.Background(), "myapp")

    // Undelete version 1
    err = kv.Undelete(context.Background(), "myapp", []int{1})

    // Permanently destroy version 2
    err = kv.Destroy(context.Background(), "myapp", []int{2})

    // Rollback to version 1 (re-writes its data as a new version)
    secret, err = kv.Rollback(context.Background(), "myapp", 1)

    // Full metadata
    metadata, err := kv.GetMetadata(context.Background(), "myapp")
    fmt.Printf("max versions: %d\n", metadata.MaxVersions)
}
```

</CodeGroup>

<Note>
  The `KVv2.Patch` method uses HTTP PATCH when the Vault server supports it (1.9+). On older servers it automatically falls back to a read-then-write approach. See `api/kv_v2.go` for the implementation.
</Note>
