## What is Vault?

Vault is a tool for securely accessing **secrets**. A secret is anything you want to tightly control access to — API keys, passwords, certificates, encryption keys, and more. Vault provides a unified interface to any secret while providing tight access control and recording a detailed audit log.

Modern systems require access to a multitude of secrets: database credentials, API keys for external services, credentials for service-oriented architecture communication, and more. Understanding who is accessing what secrets is already very difficult and platform-specific. Adding key rolling, secure storage, and detailed audit logs is nearly impossible without a custom solution. This is where Vault steps in.

## Key features

<CardGroup cols={2}>
  <Card title="Secure Secret Storage" icon="lock">
    Store arbitrary key/value pairs with encryption at rest. Vault encrypts data before writing to persistent storage — gaining access to raw storage is never enough to access your secrets.
  </Card>
  <Card title="Dynamic Secrets" icon="bolt">
    Generate secrets on-demand for AWS, SQL databases, and more. Vault creates credentials with valid permissions on demand and automatically revokes them after the lease expires.
  </Card>
  <Card title="Data Encryption" icon="shield">
    Encrypt and decrypt data without storing it. Security teams define encryption parameters and developers store encrypted data without designing their own encryption methods.
  </Card>
  <Card title="Leasing & Renewal" icon="clock">
    Every secret has a lease. At the end of the lease, Vault automatically revokes the secret. Clients can renew leases via built-in renew APIs.
  </Card>
  <Card title="Secret Revocation" icon="ban">
    Revoke not just a single secret, but entire trees of secrets — all secrets read by a specific user, or all secrets of a particular type.
  </Card>
  <Card title="Auth Methods" icon="key">
    Authenticate using tokens, AppRole, AWS IAM, GitHub, LDAP, TLS certificates, Kubernetes, and more. Vault supports multiple auth methods simultaneously.
  </Card>
  <Card title="ACL Policies" icon="shield-halved">
    Fine-grained access control with HCL-based policies. Control which paths can be read, written, deleted, and listed per role or user.
  </Card>
  <Card title="Audit Logging" icon="file-lines">
    Every interaction with Vault is logged to one or more audit devices. Tamper-evident logs with HMAC-SHA256 hashing for compliance and forensics.
  </Card>
</CardGroup>
